**Functional programming in JavaScript is easy.**

Easier, in fact, than OOP in JavaScript.

If you've been looking at academic discourses about functional programming with terms such as _referential transparency_, _lambda calculus_, and the dreaded _monad_, then it's no wonder you're confused. But you don't need to understand any of that terminology (or even the theory) to _FP_.

Here we present the fundamental **techniques** for functional programming without all the painful academic theory and terminology. You, too, can FP.

## How to FP

<ol>
  <li>
    <Link to="/reusability/" aria-label="Reusability">Functions are the basic unit of reusability</Link>
    <ul>
      <li>Functions take at least one argument</li>
      <li>Functions return exactly one result</li>
      <li>Functions can be passed as arguments to other functions</li>
      <li>Functions can be returned from other functions</li>
      <li>Functions can call functions, <em>including themselves</em></li>
      <li>Given the same arguments, functions must return the same result</li>
      <li>Don't be reaching outside the function to mess with shit</li>
      <li>Make copies when passing by reference</li>
      <li>Functions can remember things for you</li>
    </ul>
  </li>
  <li>
    <Link to="/immutability/" aria-label="Immutability">Variables are &ldquo;set and forget&rdquo;</Link>

      <li>Assign variables when they are declared</li>
      <li>Once a variable has been assigned, don't be changing it</li>
      <li>Don't erase state: create new state instead!</li>
      <li>Unreassignable !== immutable</li>
      <li>Use destructuring and the rest/spread operators to make copies</li>
    </ol>
  </li>
  <li>

      <li>Use <code>adjust</code> to update a single value in a list</li>
      <li>Use <code>append</code>, <code>insert</code>, or <code>prepend</code> to add an item to a list</li>
      <li>Use <code>concat</code> or <code>zip</code> to combine two lists</li>

      <li>Use <code>reduce</code> to combine elements in a list</li>
      <li>Use <code>reduce</code> to convert a list to some other data type</li>
      <li>Use <code>head</code> or destructuring to get the first item in a list</li>
      <li>Use <code>tail</code> or destructuring to get the list minus the first element</li>
      <li>Use <code>all</code>, <code>any</code>, or <code>none</code> to test a list</li>
      <li>Use <code>flatten</code> to flatten nested lists into a single list</li>
